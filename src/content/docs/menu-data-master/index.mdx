## Apa itu Data Master?

**Data Master** adalah kumpulan data dasar (master data) yang menjadi fondasi operasional sistem presensi RFID. Data master ini bersifat relatif statis dan digunakan sebagai referensi oleh modul-modul lain dalam sistem.

Data master mencakup informasi tentang:

- Pengguna dan peran mereka
- Instansi dan struktur organisasi
- Tahun pelajaran dan periode akademik
- Jurusan, kelas, dan mata pelajaran
- Jabatan dan posisi pegawai
- Jadwal presensi harian
- Informasi dan pengumuman

<Aside type="note" title="Penting!">
  Data master harus diisi dengan lengkap dan akurat sebelum sistem dapat
  beroperasi secara optimal. Data ini akan menjadi acuan untuk seluruh transaksi
  presensi.
</Aside>

## Siapa yang Dapat Mengakses?

Menu Data Master memiliki kontrol akses yang ketat untuk menjaga integritas data:

| Peran           | Akses          | Hak Akses                                      |
| --------------- | -------------- | ---------------------------------------------- |
| **Super Admin** | Full Access    | Create, Read, Update, Delete semua data master |
| **Operator**    | Limited Access | CRUD untuk sebagian besar data master          |
| **Wali Kelas**  | Read Only      | Hanya bisa melihat data kelas yang diampu      |
| **Pegawai**     | No Access      | Tidak memiliki akses ke data master            |
| **Siswa**       | No Access      | Tidak memiliki akses ke data master            |

<Aside type="caution" title="Keamanan Data">
  Hanya pengguna dengan peran Super Admin dan Operator yang dapat mengelola data
  master. Pastikan kredensial admin dijaga dengan baik.
</Aside>

## Pentingnya Data Master

### Dampak Data Master pada Sistem

<Steps>

1. **Akurasi Presensi**

   Data jadwal yang tepat memastikan presensi tercatat dengan benar (hadir/terlambat)

2. **Notifikasi yang Relevan**

   Data kontak yang akurat menjamin notifikasi sampai ke penerima yang tepat

3. **Laporan yang Valid**

   Data master yang lengkap menghasilkan laporan yang akurat dan dapat dipercaya

4. **Efisiensi Operasional**

   Data terstruktur mempercepat proses kerja harian administrator

5. **Keamanan Sistem**

   Role management yang baik melindungi data sensitif dari akses tidak sah

</Steps>

### Risiko Data Master yang Tidak Akurat

<CardGrid>
  <Card title="Presensi Salah" icon="warning">
    Jadwal tidak tepat → siswa hadir dicatat terlambat
  </Card>

<Card title="Notifikasi Gagal" icon="error">
  Nomor tidak valid → orang tua tidak dapat notifikasi
</Card>

<Card title="Laporan Tidak Valid" icon="ban">
  Data tidak lengkap → laporan tidak dapat dihasilkan
</Card>

  <Card title="Keamanan Lemah" icon="lock">
    Role tidak teratur → akses data oleh pihak tidak berwenang
  </Card>
</CardGrid>

## Urutan Pengisian Data

Untuk setup sistem yang optimal, ikuti urutan pengisian data master berikut:

<Steps>

1. **Setup Peran (Roles)**

   Buat peran pengguna terlebih dahulu: Super Admin, Operator, Wali Kelas, Pegawai, Siswa

   [Panduan Peran →](/menu-data-master/peran/)

2. **Tambahkan Pengguna Admin**

   Buat akun untuk administrator dan operator sistem

   [Panduan Pengguna →](/menu-data-master/pengguna/)

3. **Input Data Instansi**

   Lengkapi profil sekolah/madrasah termasuk logo dan identitas

   [Panduan Instansi →](/menu-data-master/instansi/)

4. **Setup Tahun Pelajaran**

   Tentukan tahun ajaran aktif dan periode akademik

   [Panduan Tahun Pelajaran →](/menu-data-master/tahun-pelajaran/)

5. **Tambahkan Jabatan**

   Buat daftar jabatan untuk pegawai (Guru, Staff, dll)

   [Panduan Jabatan →](/menu-data-master/jabatan/)

6. **Input Jurusan**

   Tambahkan program studi yang tersedia

   [Panduan Jurusan →](/menu-data-master/jurusan/)

7. **Buat Kelas**

   Setup rombongan belajar per tingkat dan jurusan

   [Panduan Kelas →](/menu-data-master/kelas/)

8. **Tambahkan Mata Pelajaran**

   Input daftar mapel yang diajarkan

   [Panduan Mata Pelajaran →](/menu-data-master/mata-pelajaran/)

9. **Atur Jadwal Presensi**

   Tentukan waktu masuk/pulang dan toleransi keterlambatan

   [Panduan Jadwal Presensi →](/menu-data-master/jadwal-presensi/)

10. **Buat Informasi/Pengumuman (Opsional)**

    Tambahkan pengumuman untuk broadcast ke pengguna

    [Panduan Informasi →](/menu-data-master/informasi/)

</Steps>

<Aside type="tip" title="Tips Setup Awal">
  - Siapkan semua data di Excel terlebih dahulu - Gunakan fitur import untuk
  mempercepat input data massal - Verifikasi setiap data sebelum melanjutkan ke
  step berikutnya - Backup data secara berkala
</Aside>

## Troubleshooting Data Master

### Masalah Umum

<Aside type="caution" title="Data Tidak Bisa Dihapus">
  **Penyebab:** Data sedang digunakan oleh modul lain **Solusi:** - Cek relasi
  data dengan modul lain - Nonaktifkan data (jangan hapus) - Arsipkan data lama
  - Hubungi admin jika perlu force delete
</Aside>

<Aside type="caution" title="Import Data Gagal">
  **Penyebab:** Format file atau data tidak sesuai template **Solusi:** -
  Download template yang benar - Periksa format setiap kolom - Hapus data
  duplikat - Pastikan tidak ada cell yang kosong di kolom wajib
</Aside>

<Aside type="caution" title="Perubahan Tidak Tersimpan">
  **Penyebab:** Session expired atau error validasi **Solusi:** - Login ulang -
  Cek pesan error validasi - Isi semua field yang required - Clear browser cache
  dan coba lagi
</Aside>

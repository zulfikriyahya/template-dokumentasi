## Pengertian Instansi

**Instansi** adalah data profil lengkap sekolah/madrasah yang menggunakan sistem presensi RFID. Data ini menjadi identitas institusi yang akan muncul di berbagai laporan, dokumen, dan notifikasi sistem.

### Pentingnya Data Instansi

<CardGrid>
  <Card title="Identitas Resmi" icon="home">
    Menjadi identitas institusi di semua dokumen dan laporan resmi
  </Card>

<Card title="Jadwal Operasional" icon="clock">
  Menentukan hari kerja dan jadwal presensi otomatis berdasarkan status
</Card>

<Card title="Laporan Profesional" icon="document">
  Data lengkap untuk header laporan, surat, dan dokumen resmi
</Card>

  <Card title=" Branding Konsisten" icon="star">
    Logo dan identitas visual yang konsisten di seluruh sistem
  </Card>
</CardGrid>

## Troubleshooting

<Tabs>
  <TabItem label="Logo Tidak Muncul">
    **Kemungkinan Penyebab:**
    - File terlalu besar (>2MB)
    - Format tidak didukung
    - Upload error
    - Cache browser
    
    **Solusi:**
    1. Kompres logo ke < 500KB
    2. Gunakan PNG atau JPG
    3. Upload ulang dengan koneksi stabil
    4. Clear browser cache (Ctrl+Shift+R)
    5. Cek apakah logo tersimpan di server
  </TabItem>
  
  <TabItem label="Jadwal Tidak Sesuai Status">
    **Gejala:**
    Institusi Negeri tapi laporan 6 hari atau sebaliknya
    
    **Penyebab:**
    - Status salah di setup awal
    - Ada jadwal custom yang override
    - Cache jadwal belum refresh
    
    **Solusi:**
    1. Verifikasi status instansi
    2. Cek jadwal presensi custom
    3. Nonaktifkan jadwal yang konflik
    4. Refresh sistem (logout-login)
    5. Rekalkulasi laporan
  </TabItem>
  
  <TabItem label="Data Tidak Bisa Diubah">
    **Gejala:**
    Form edit disabled atau error saat save
    
    **Penyebab:**
    - User bukan Super Admin
    - Data sedang digunakan proses lain
    - Validation error
    - Database lock
    
    **Solusi:**
    1. Pastikan login sebagai Super Admin
    2. Tunggu proses lain selesai
    3. Cek error message untuk detail
    4. Isi semua field required
    5. Hubungi tech support jika persist
  </TabItem>
</Tabs>
